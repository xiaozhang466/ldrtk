# 耘小智 01 Web UI 设计说明

本文记录当前 Web 前端的页面结构和交互约定。实现以 Ant Design 5 为基础，地图和点云区域使用 Cesium、Three.js、Canvas 等组件承载。

## 设计原则

- 面向现场操作：优先保证状态清晰、按钮明确、重复操作效率高。
- 页面保持工具化：避免营销式大段说明，功能入口直接可用。
- 地图/点云区域优先保证可视面积，控制项集中在顶部或侧栏。
- 危险操作需要确认，运行中状态需要持续反馈。

## 页面结构

### 首页 `/`

- 顶部展示产品和连接状态。
- 状态卡片展示 ROS、定位、电池等关键信息。
- 功能入口为地图管理、导航、设置；路径规划从地图管理内的操作进入（路由 `/path-planning`）。

### 地图管理 `/maps`

当前地图相关能力集中在该页面，通过 Tab 切换：

- 地图管理：地图列表、创建、删除、重命名、切换、坐标更新。
- 建图控制：启动/停止建图，显示建图状态和实时点云。
- PCD 预览：选择地图和 PCD 文件，Three.js 显示点云，支持着色、过滤、视角适配。

### 路径规划 `/path-planning`

- 按地图类型选择本地 Canvas、GPS 地图或融合地图视图。
- 支持路径点创建、编辑、删除和保存。
- 融合/GPS 地图使用 `gps_origin` 进行经纬度与局部米制坐标换算。

### 导航 `/nav`

- 选择地图与路径。
- 展示路径点、机器人位置和导航状态。
- 下发开始、暂停、继续、停止等任务控制。

### 设置 `/settings`

- 基础配置、RTK、建图、导航相关设置入口。
- ROS 连接状态应始终可见。

## 地图视图约定

- 本地点云：`LocalMapView` / `MapPreview`，使用 Three.js 和 PCDLoader。
- GPS 地图：`GPSMapView` + `MarsMapForManager`，使用 Cesium 天地图。
- 融合地图：`FusionMapView` / `FusionMapForManager`，在天地图上叠加 PCD 栅格图层。
- 本地栅格路径规划：`SimpleMapCanvas`，基于 `map.yaml` 的 `resolution` 和 `origin` 做像素/世界坐标换算。

## 已移除的旧设计

- 独立 `/map` 假数据页面。
- 独立 `/pcd-viewer` 和 `/mapping-control` 页面，功能已合并到 `/maps`。
- 旧 `/gps-map-create` 多点配准流程。
- Mars2D 方案说明，当前地图预览主要使用 Cesium。

## 响应式要求

- 桌面端优先，常见目标分辨率为 1366x768 和 1920x1080。
- 关键按钮和选择器不能因内容变化挤压地图区域。
- 表格和点云面板需要在低高度屏幕下保留滚动能力。
